# AWS Root Access 


When accessing AWS with the root user, the following MFA (Multi-Factor Authentication) options are available:

**Authenticator Apps (TOTP)**
Time-based one-time passwords generated by apps like Google Authenticator, Authy, or Microsoft Authenticator. The app generates a 6-digit code that refreshes every 30 seconds.

**Hardware Security Keys (FIDO2/WebAuthn)**
Physical devices like YubiKey or other FIDO2-compliant keys. You plug in or tap the device to authenticate. These are the most phishing-resistant option and are strongly recommended for root users.

**Hardware TOTP Tokens**
Dedicated physical devices (e.g., Gemalto tokens) that generate time-based codes, similar to authenticator apps but on standalone hardware — no smartphone needed.

---

**AWS recommendations for root users specifically:**

- AWS **strongly recommends** using a **hardware security key** for root, since root has unrestricted account access.
- You can register **up to 8 MFA devices** on a root account, allowing you to combine types (e.g., a hardware key + an authenticator app as backup).
- SMS-based MFA is **not available** for root users (it was deprecated by AWS).

|---|---|---|---|

For a root account, the **hardware security key** is the gold standard due to its phishing resistance and the critical nature of root access.  


# Hardware Security Key  

Here's how to set up a **Hardware Security Key (FIDO2/WebAuthn)** for your AWS root user:

---

## What You Need
- A **FIDO2-compatible key** (e.g., YubiKey 5, Google Titan Key)
- Access to your AWS root account
- A supported browser (Chrome, Firefox, Edge, Safari)

---

## Step-by-Step Setup

**1. Sign in as Root**
- Go to [console.aws.amazon.com](https://console.aws.amazon.com)
- Sign in with your root email and password

**2. Go to Security Credentials**
- Click your account name (top-right)
- Select **"Security credentials"**

**3. Find the MFA Section**
- Scroll down to **"Multi-factor authentication (MFA)"**
- Click **"Assign MFA device"**

**4. Choose Security Key**
- Enter a device name (e.g., "YubiKey-Root")
- Select **"Security key"** as the MFA type
- Click **Next**

**5. Register the Key**
- AWS will prompt your browser to detect the key
- **Insert your hardware key** into a USB port (or use NFC if supported)
- **Touch/tap the key** when the light flashes — this confirms physical presence
- The key registers automatically

**6. Done**
- Click **"Close"** — your key is now active

---

## Logging in With the Key

Next time you sign in as root:
1. Enter email + password as usual
2. When prompted for MFA → **insert and touch your hardware key**
3. Authentication completes automatically — no code to type

---

## Best Practices

- 🔑 **Register 2 hardware keys** — one primary, one backup stored securely (e.g., a safe)
- 🚫 **Never store the key with the laptop** — defeats the purpose
- 📦 **Store the backup key offsite** if possible
- 🔒 Root should be used **as rarely as possible** — prefer IAM users/roles for daily work

---
