The security tradeoff — same secret, two locations
Storing password and TOTP in the same vault reduces 2FA's protection — if the vault is compromised, both factors are exposed
Mitigation: use a very strong vault master password + key file, and keep TOTP in a separate app for high-stakes accounts like AWS root
For most personal accounts (Google, GitHub, Amazon) same-vault TOTP is a practical, acceptable tradeoff vs. losing access
What happens if you lose your phone?
No backup (e.g. Google Authenticator, no sync)
TOTP codes are gone. You must use recovery codes for every account, then re-register TOTP from scratch on a new device. High risk of lockout.
With backup (Authy / 1Password / KeePassXC vault copy)
Log in to your backup service on a new phone. All secrets restore immediately. No recovery codes needed, no re-scanning QR codes.
Recommendation by situation
Already using KeePass
Use KeePassXC — TOTP built-in, everything in one encrypted vault
Want best backup
Use Authy — encrypted cloud sync, survives phone loss with no recovery codes needed
Simplest setup
Google Authenticator — familiar, widely supported; enable Google account sync to avoid lockout
AWS root specifically
Hardware key is better — but KeePassXC or Authy work if hardware is not an option
Always save backup codes: Every site that supports TOTP also offers one-time recovery codes at setup. Save them offline (password manager or printed copy in a safe). If your phone is lost and you have no backup, these are your only way back in.
