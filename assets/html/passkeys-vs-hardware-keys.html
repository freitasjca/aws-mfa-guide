Same cryptography — different threat model
Both use public-key cryptography — private key signs a challenge, server verifies with public key
Both are domain-bound — immune to phishing by a fake site
Passkey: if your device or cloud account is compromised, the private key could theoretically be extracted
Hardware key: private key lives in a tamper-proof chip — physically cannot be exported, ever

Key insight: Passkeys trade the hardware key's "cannot be copied" guarantee for the convenience of cloud sync and no physical device. For most accounts, that tradeoff is worth it. For AWS root, it isn't.
